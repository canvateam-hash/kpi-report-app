# KPIレポート作成システム

ANALYSIS_KPI.md をベースに、実データ検証・ユーザー確認を重ねて構築したKPI自動計算Webアプリ（Streamlit製）。

**このアプリは `app.py` 1ファイルだけで動作します。** フォルダ構造を気にする必要はありません。

## 実装状況：①〜⑫ 全項目完成

| KPI | 内容 | 必要データ |
|---|---|---|
| ① | 初回離脱率 | 受注データ |
| ② | 期間別解約率 | 受注データ |
| ③ | F2転換率 | 定期継続率データ（全体/縛りあり/縛りなし） |
| ④ | 定期継続率 | 定期継続率データ（全体/縛りあり/縛りなし） |
| ⑤⑥⑦ | LTV・平均購入回数・アップセル率 | 受注データ |
| ⑧ | 顧客推移データ | 受注データ＋定期受注データ |
| ⑨ | 稼働顧客数 | 受注データ＋基準日 |
| ⑩ | RF分析 | 受注データ＋基準日 |
| ⑪ | 解約理由 | 定期受注データ |
| ⑫ | スコアリング（顧客分類） | 受注データ |

## 起動方法

1. `app.py` と `requirements.txt` を**同じフォルダ**にダウンロード
2. フォルダに移動してターミナル（コマンドプロンプト）を開く
3. 必要なライブラリをインストール
   ```bash
   pip install -r requirements.txt
   ```
4. アプリを起動
   ```bash
   python -m streamlit run app.py
   ```
5. ブラウザで `http://localhost:8501` が自動的に開きます

`streamlit` コマンドが見つからないと言われた場合は、`python -m streamlit run app.py` の形（`python -m` を付ける）で実行してください。

## 使い方
1. 左のサイドバーで、必要なデータを**まとめて一度に**アップロード
   - 受注データCSV（複数月分まとめてOK）
   - 定期受注データCSV（複数月分まとめてOK）
   - 継続率データCSV（全体／縛りあり／縛りなしの3種類）
   - 基準日（⑨⑩で使用）
2. アップロードすると、上部に①〜⑫のタブが並びます。タブを自由に切り替えるだけで、再アップロード不要ですべてのKPIを閲覧できます
3. 各KPIに必要なデータがアップロードされていない場合は、そのタブ内に案内が表示されます（他のタブの表示には影響しません）
4. 結果はCSV／Excelでダウンロード可能

## 必須列
- **受注データCSV**：`顧客番号` `受注日` `定期回数` `対応状況` `決済状況` `合計`
  （②⑤⑦⑨は追加で `発送日` `発送予定日` `更新日` `お約束回数（定期）` `次回発送予定日` を使用）
- **定期受注データCSV**：`顧客番号` `ステータス` `作成日` `定期回数`
  （⑧は追加で `停止日` `キャンセル日`、⑪は `キャンセル理由` を使用）
- **継続率データCSV**：`定期受注作成日（年月）` `n~n+1回` `継続率 (%)` `継続予定率 (%)` `合計 (件)` `売上済 (件)` `売上前 (件)` `待機中 (件)` `離脱 (件)`

列名が実データと異なる場合はエラーメッセージで通知されます。

## 主要ロジックの要点

### スコア方式（①②⑤⑨で共通）
対応状況スコア（0/1）＋決済状況スコア（0/1）＝0・1・2のいずれか。
- ①の初回購入者（分母）＝スコア≥1、離脱者（分子）＝スコア≤1（スコア1は両方に計上）
- ⑤⑨等の「完全購入」＝スコア=2のみ

### 真の初回購入行の抽出（①②⑤で共通）
定期回数=1かつ受注月が登録月と一致する行のみを「真の初回購入」として抽出
（再定期による定期回数リセットを誤カウントしない）。

### ⑦アップセル判定
SKUコードの静的リストではなく「お約束回数（定期）＞0」を直接判定基準に使用
（新商品追加時も自動対応、③④の縛りあり/なし判定とも一貫性を確保）。

### ⑧アクティブ判定
定期受注データの「作成日／停止日／キャンセル日」から、対象月末時点で有効な契約を
1件以上持つ顧客を「アクティブ」と判定。
